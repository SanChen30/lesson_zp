## 构造函数

1. constructor 构造函数

new 的方式实例化对象，this 指向实例化的对象

每个实例自己的属性和方法，都写在构造函数中，而不是在原型对象上。

prototype 原型对象上的属性和方法，让这个类的所有实例化对象都可以访问到，是共享的。

虽然不是直接设置在构造函数中，但是可以通过实例访问到原型对象上的属性和方法。

JS 面向对象是原型式的面向对象，哲学性的面向对象，不是基于类的面向对象。

2. 何为prototype?

1) JS 为了完成原型式的面向对象构建，实例的属性用constructor，用prototype 属性设置共有的方法和属性

每个函数都有一个 prototype 属性，指向一个对象，包含了所有实例化对象共有的属性和方法。

2) 传统的class 面向对象是血缘关系的，class 定义属性和方法的模板（抽象），实例是具体的

js 原型式的面向对象，不是血缘关系的，而是基于原型的关系。

3) 所有的实例化对象都有一个 __proto__ 私有属性，指向它的原型对象。

4) 原型对象上有一个 constructor 属性，指向它的构造函数。

由这个构造函数创建的实例，可以共享原型对象上的属性和方法。

3. 任何对象都默认指向 Object.prototype，toString() 方法是 Object.prototype 上的方法，都会打印 [object Object]，除非重写了 toString() 方法。

4. Object.prototype 的原型是 null，停止查找

null 表示没有原型对象
