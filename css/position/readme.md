# 定位

- 离开文档流

文档流是 HTML 元素默认的布局方式，块级元素垂直排列，行内元素水平排列，遵循从上到下，从左到右的自然顺序

元素在文档流中，如何离开？

- `position`: `static | relative | absolute | fixed | sticky`


- `relative`：相对定位元素，是相对于元素在原来文档流中的位置进行偏离，
            不会脱离文档流，原来在标准流的位置继续占有，只是视觉上偏离了原来的位置，后面的盒子仍然以标准流的方式对待它。

- `absolute`：绝对定位元素，是相对于最近的非 `static` 定位祖先元素进行定位的。
            如果没有非 `static` 定位的祖先元素，那么就相对于文档.body 元素。
            绝对定位元素会脱离文档流，原来在标准流的位置会被释放，后面的盒子会忽略它的存在，

- `display:none`; 隐藏元素，不占用空间，不会影响到其他元素的布局

- `opacity`:0; 隐藏元素，占用空间，会影响到其他元素的布局

- `fixed`：固定定位元素，是相对于浏览器窗口进行定位的。
        固定定位元素会脱离文档流，原来在标准流的位置会被释放，后面的盒子会忽略它的存在，
        但是固定定位元素会固定在浏览器窗口中，不会随着滚动条的滚动而滚动。
  
- `sticky`：是 CSS 中一种特殊的定位方式，结合了相对定位（`relative`）和固定定位（`fixed`）的行为。
  
    元素在默认状态下处于正常文档流中，表现如同 `position: relative`，会占据其原本的空间；

    但当页面滚动到由 `top`、`bottom`、`left` 或 `right` 指定的阈值时，该元素就会“粘”在视口（或最近的可滚动祖先容器）的对应位置，表现得像 `position: fixed`。    

    需要注意的是，粘性定位的效果仅在其父容器的内容边界内有效，且必须显式设置至少一个偏移属性（如 `top: 0`）才能生效。

    如果父容器设置了 `overflow: hidden`、`scroll` 或 `auto`，那么 `position: sticky` 的“粘性”效果将被限制在这个父容器内部，而不会相对于整个浏览器视口生效。

  `sticky` 元素的定位是相对于最近的可滚动祖先容器，而不是浏览器视口。   

- `static`：静态定位元素，是元素的默认定位方式，不会被其他定位属性所影响，也不会脱离文档流。
            元素按照正常的文档流进行定位，用它来取消元素的定位属性，将元素恢复到文档流中的原始位置。
