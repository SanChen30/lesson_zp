    每一方的发送和接收要2次，握手四次
    第二次握手：客户端知道：服务端能收（因为发了 ACK），服务端能发（因为发了 SYN）。
               服务端知道：客户端能发（因为收到了 SYN），但还不知道客户端能不能收（需要第三次握手确认）。
