## 架构图

- 操作系统考点

进程 = 操作系统分配资源的最小单位，独立的内存空间，一个正在运行的程序实例
线程 = CPU 调度的最小单位，共享进程内存，进程里的执行路径，一个进程可以包含多个线程

 - 浏览器是多进程架构，整个过程需要各个进程之间的配合
   - 浏览器主进程
     - 提供用户的操作反馈
     location 输入 -> 浏览器进程 -> 渲染进程 -> 页面展示
     - 浏览历史入栈新的记录
     - 请求状态 loading 的显示
        网络请求开始，显示 loading 图标
        请求完成 loading 图标隐藏
        请求的资源交给渲染进程
     - 管理子进程，进程间的通信 **ipc**
        Inter-Process Communication
     - 当前已经显示或者重新刷新
        beforeunload 事件
    浏览器进程负责用户交互、子进程管理和文件存储（缓存、cookie、localStorage）
    - 网络进程是面向渲染进程和浏览器进程等提供网络下载功能


1. 首先，浏览器进程接受到用户输入的URL请求，浏览器进程将该URL转发给网络进程
   - URL 补全标准URL 带上协议头（http(s)://www.
      我输入 chatgpt.com -> https://chatgpt.com/
      我输入 baidu.com -> https://www.baidu.com/
   - 如果是搜索关键词，去向默认搜索引擎URL带上关键词
   - 网址 英文、中文 可读性好   DNS 解析
      OSI 七层协议  通过 IP 地址来通信
      拿着域名（domain）去查询 IP 地址
      key: domain
      value: IP 地址
      DNS 是一个分布式的数据库
      - 本地 DNS 缓存查找
      - 局域网查找
      - 根域名服务器查找
      - 中国根服务器（海底光缆）
      - 美国服务器（顶级服务器）
  - 如果本地有缓存，直接返回缓存的结果，不发送请求
      text/css   image/jpeg
  - 三次握手 确保双方都具有请求发送的能力
  - 发送请求行和请求头
      GET URL http版本
      请求头 jwt token Authorization: Bearer <token>
      cookie 
  - 发送响应头，解析响应头数据，并将数据转发给浏览器进程
      Content-Type 浏览器做出不一样的响应
      text/html -> 渲染进程准备接收
      text/css image 下载
      doc mp3
      状态码 
  - 浏览器接收到“提交导航”的消息后，开始准备接收HTML数据，接收数据的方式是直接和网络进程建立数据管道（pipe）
  - 最后渲染进程会向浏览器进程“确认提交”，已经准备好接收和解析页面数据
  - 浏览器进程接收到渲染进程“提交文档”消息后，便开始移除之前旧的文档，然后更新浏览器进程中的页面状态，进入loading状态

  用户发出 URL 请求到页面开始解析的整个过程，就叫导航
