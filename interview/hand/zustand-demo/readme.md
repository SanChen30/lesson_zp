# 手写zustand

代码能力

- zustand 状态管理

- 为什么需要它？
    用于组件通信，随着层次的叠加，传递复杂，需要一个中央管理来共享状态
    管好状态（财务），做到状态共享
    简单
    非常流行 star 数比较多
- redux 老牌的react状态管理库
    复杂
- 读过源码，能够手写
- 核心内容
    - 以store 提供状态
    - 修改状态
    - 响应更新 订阅发布者模式
    - 自定义hook
- 手写要实现以下功能
    - 实现 create 高阶函数，接收一个函数（set, get, store），
        返回一个状态对象（状态的初始化），里面包含状态、修改状态的方法
        返回一个 store hook 函数，useXxxStore
    - useXxxStore 接收一个函数，返回某一些状态或方法，
        为空，整体返回
    - subscribe 方法
        添加一个订阅者，在状态发生改变的时候执行，响应式

## 开始实现
- 实现一个最简单的 Store
    存储以及管理状态，怎么存，怎么取，怎么改
- 加入订阅机制
    抖音的网红（store 发布者）和订阅者（订阅关系）
    以后发布者发布内容（状态改变）
    订阅者收到消息，并执行
    - Store 对象 subscribe 方法
        接收一个函数
    - state 改变的时候，执行这个函数
- 支持 setState 接收函数，会将之前的状态给它
- 支持状态合并
    Object.assign()
- create 比 createStore 更好理解，减少学习成本
- 为什么要学习源码
    - 提升自己的代码能力，学到高级的写法
    - 了解框架的细节，用好它
        zustand 性能，useXxxStore 制定需要的状态
